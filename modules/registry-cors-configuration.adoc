== Configuring {registry} Cross-Origin Resource Sharing (CORS)

[role="_abstract"]
To allow your browser-based application (React, Angular, Vue, or plain JavaScript) to call the {registry} REST API from a different origin, you must configure Cross-Origin Resource Sharing (CORS).
Without the correct CORS configuration on the {registry} server, the browser blocks API calls from your application, even though the same requests succeed from a non-browser client such as `curl`.
{registry} includes built-in CORS support based on the Quarkus HTTP CORS filter that you can configure for your deployment.
